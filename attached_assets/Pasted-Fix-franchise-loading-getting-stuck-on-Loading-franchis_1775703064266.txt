Fix franchise loading getting stuck on "Loading franchise..."

ROOT PROBLEM:
The app is stuck because loadFromSupabase() is not receiving a valid sim_state.
We must guarantee that:
1. sim_state is ALWAYS written after create
2. loadFromSupabase() ONLY resolves when sim_state exists
3. UI does not hang indefinitely

REQUIRED FIXES:

1. ENFORCE CREATE FLOW COMPLETION

Inside handleCreate / createFranchise flow:

- After initSeason(), IMMEDIATELY persist sim_state:

await supabase
  .from('franchise_seasons')
  .update({
    sim_state: generatedSeason,
    updated_at: new Date().toISOString()
  })
  .eq('franchise_id', franchiseId)

- Add console.log:
console.log("✅ sim_state written", franchiseId)

This guarantees the season exists before any load.

---

2. FIX loadFromSupabase()

Rewrite loadFromSupabase() to:

- Poll ONLY for rows where sim_state IS NOT NULL
- Retry up to 8 times (3 sec interval)
- If found → setSeason and STOP syncing
- If NOT found → fail cleanly

Example:

let attempts = 0

while (attempts < 8) {
  const { data } = await supabase
    .from('franchise_seasons')
    .select('*')
    .eq('franchise_id', franchiseId)
    .not('sim_state', 'is', null)
    .single()

  if (data?.sim_state) {
    console.log("✅ Season loaded")
    setSeason(data.sim_state)
    setIsSyncing(false)
    return
  }

  console.log("⏳ Waiting for sim_state...", attempts)
  await new Promise(r => setTimeout(r, 3000))
  attempts++
}

console.error("❌ Season failed to load")
setIsSyncing(false)
setSeason(null)

---

3. PREVENT INFINITE SPINNER

In UI:

IF isSyncing && !season:
→ show loading screen

IF !isSyncing && !season:
→ show error state:
"Failed to load franchise. Please retry."

DO NOT allow spinner forever.

---

4. ADD DEBUG LOGGING (CRITICAL)

Add logs at:

- createFranchise start/end
- initSeason complete
- sim_state write success
- loadFromSupabase start
- each polling attempt
- successful load
- failure

---

SUCCESS CRITERIA:

- After creating a franchise, entering HQ ALWAYS loads a season
- Spinner disappears within ~5–20 seconds
- If something fails, user sees an error instead of infinite loading
- Console clearly shows where failure occurs

Do NOT change unrelated UI or features.
Fix only the create → persist → load pipeline.
